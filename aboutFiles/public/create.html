Introdução:



É usada uma pequena linguagem de marcação criada por mim mesmo chamada: listdata.
O interpretador atual coompreende um total de 7 tags, sendo duas delas tags de acoplamento, e o resto são tags de conteudo.
Todas as Tags são declaradas com &{nome da tag}&, e devem ser fechadas com ${nome da tag}$, exemplo: &DATA&Conteudo$DATA$

Tag Função Tipo
HEADER Engloba todas as meta tags usado internamente pelo programa Acoplamento
CONTENT Engloba todos os dados visualizados e manipulados Acoplamento
USER Autor do arquivo Conteudo
NAME Nome do arquivo Conteudo
PUBLIC Diz se o arquivo é publico ou não. Recebe apenas valores booleanos (true ou false).
Use apenas false se os seus dados forem cifrados, você pode fazer a cifragem manual usando os comandos de terminal		 Conteudo
TYPE Diz qual é a estrutura de dado do arquivo.
Tipos disponiveis: FIFO, LIFO, LISTA		 Conteudo
DATA É onde irá ficar um dado da lista Conteudo
Exemplo:

&HEADER&
  &USER&Erik$USER$
  &NAME&muito$NAME$
  &PUBLIC&true$PUBLIC$
  &TYPE&LISTA$TYPE$
$HEADER$

&CONTENT&
  &DATA&eu$DATA$
  &DATA&tu$DATA$
  &DATA&ele$DATA$
  &DATA&nos$DATA$
  &DATA&vos$DATA$
  &DATA&eles$DATA$
$CONTENT$

Estrutura e Regras:



* Todo arquivo deve contar as duas tags de acoplamento "HEADER" e "CONTENT".
* Na Tag "HEADER" é necessario ter todas as seguintes tags: "USER, "NAME", "PUBLIC", "TYPE".
* Se for usar "false" para a tag "PUBLIC" você deve ter em conciencia que as seguintes tags deverão ser cifradas conforme a chave de login (é possivel gerar uma automaticamente pelos Comandos de terminal): "USER" e todas as tags "DATA"
* Cada tag "DATA" dentro de "CONTENT" é uma nova linha de dado
